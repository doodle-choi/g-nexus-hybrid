# Phase 1: High Fidelity UI Draft Plan (보고용 목업 중심)

사용자(보고 대상자)에게 백엔드나 데이터 연동 이전에 전체적인 "그림(Look and Feel)"을 확실하게 보여주기 위한 전략입니다. 서버/DB와의 연동을 철저히 배제하고 정적(Mock) 데이터만을 활용하여 Phase 1을 시각적 완성도 위주로 재구성합니다.

## 1. 진행 목표
- **빠른 시각화:** 실제 데이터베이스나 API 연결 없이 완벽한 정적(Static) UI를 구성하여 보고용 청사진 마련.
- **레이아웃 완성:** G-Nexus의 메인 테마(Samsung Blue & Slate Gray)가 적용된 GNB(글로벌 내비게이션 바), 좌측 사이드바, 공통 레이아웃 구조의 완성.
- **핵심 화면 목업(Mockup):** 
  1. **Executive Dashboard (임원용 대시보드):** 트렌드 카드, 차트(Recharts 활용 더미 데이터), 상태 뱃지 등 즉각적인 Insight를 주는 화면.
  2. **Data Operations (실무자용 데이터 보드):** TanStack Table을 활용한 밀도 높은 표(Grid) UI 프레임워크 뼈대 (더미 데이터 삽입).
  3. **Data Ingestion (데이터 업로드 화면):** 파일 드래그 앤 드롭 존 및 매핑 설정 UI 형태 구성.

## 2. 작업 순서 (Action Plan)

1. **설계 (Layout Architecture):** 
   - `Sidebar.tsx`, `Header.tsx` 등 핵심 공통 레이아웃 컴포넌트를 분리.
   - `lucide-react` 아이콘을 활용한 전문적인 메뉴 구성(대시보드, 데이터 수집, 데이터 매핑, 설정 등).
2. **더미 데이터 구축 (Mock Data):**
   - 시각적으로 그럴듯한(Realistic) JSON 형태의 더미 데이터를 프론트엔드 단에 하드코딩.
   - 예: 최근 6개월간 비용 트렌드, 품질 알림 건수, 테이블 형태의 품질 이슈 리스트.
3. **핵심 페이지 UI 마크업:**
   - **Dashboard (Home):** Recharts 라이브러리를 설치 후 꺾은선/막대 그래프 목업 배치.
   - **Data Grid (Table):** shadcn Table 또는 TanStack Table 기본 렌더링.
4. **리뷰 및 수정:** 구성된 화면을 브라우저에 띄우고 라이브 피드백을 통해 톤앤매너와 구조를 다듬습니다.

## 3. 의견 요청
이 방향성에 동의하신다면 바로 `task.md`를 업데이트하고, 전체 화면 레이아웃(사이드바/헤더)과 대시보드 그래프 목업 작업에 착수하겠습니다. (보고용 시연을 위해 필요한 특정 메뉴 이름들이나 지표 이름들이 있으시다면 말씀해 주셔도 좋습니다.)
