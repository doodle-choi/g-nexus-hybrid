# G-Nexus 하이브리드 - 전역 레이아웃 설계안 (Global Layout Design)

G-Nexus의 목적(품질, 서비스, 비용 데이터의 통합 시각화 및 인사이트 도출)과 주요 타겟층(임원진 및 실무진)을 고려할 때, **'고밀도 데이터 처리'**와 **'직관적인 내비게이션'**을 동시에 만족해야 합니다. 이를 위해 최신 엔터프라이즈 SaaS에서 널리 쓰이는 **"Collapse형 좌측 사이드바 + 기능 집약형 상단 헤더"** 구조를 제안합니다.

---

## 1. 전체 화면 구조 (Grid Layout)

화면은 크게 3가지 영역(Header, Sidebar, Main Content)으로 분할되며, 불필요한 Footer는 제거하여 데이터 표시 영역(Main Content)을 극대화합니다.

```
+-------------------------------------------------------------+
| [Logo] G-Nexus     [Search]     [Global Filters(Date)] [User]| <-- 상단 헤더 (Header)
+-------+-----------------------------------------------------+
|       |                                                     |
| [Nav] |                                                     |
| Home  |                                                     |
|       |                                                     |
| [Nav] |                   Main Content Area                 | <-- 메인 콘텐츠 (Dashboard / Data Grid)
| Data  |                                                     |
|       |                                                     |
| [Nav] |                                                     |
| Map   |                                                     |
|       |                                                     |
+-------+-----------------------------------------------------+
  ^ 좌측 사이드바 (Sidebar)
```

---

## 2. 각 영역별 상세 구성 및 역할

### 🔴 2-1. 좌측 사이드바 (Sidebar Navigation)
화면의 좌측에 위치하며 주된 메뉴 이동을 담당합니다. 공간 절약을 위해 접기/펴기(Collapse) 기능을 제공해야 합니다.

- **메인 카테고리 (Primary Nav):**
  - 📊 **Executive Dashboard (대시보드):** 메인 화면. 핵심 지표(KPI) 트렌드 요약 및 알림 상태 표시.
  - 📋 **Data Operations (데이터 보드):** TanStack Table(또는 AG Grid) 기반의 고밀도 수정/열람 그리드. (서브 메뉴: 품질 데이터, 비용 데이터, CS 데이터 등)
  - 📥 **Data Ingestion (데이터 적재):** 외부 JSON/Markdown 데이터 파일 업로드(Drag&Drop) 및 현황 모니터링.
  - 🔗 **Data Mapping (매핑 설정):** 비정형 데이터를 시스템 표준(Schema)에 맞게 드래그 앤 드롭으로 연결하는 규칙 설정 메뉴.
- **유틸리티 카테고리 (Bottom Nav):**
  - ⚙️ **Settings:** 시스템 환경 설정, 테마, SSO 연동 등 관리자 기능.

### 🔵 2-2. 상단 헤더 (Global Top Header)
사이드바는 정적인 메뉴 이동을 담당한다면, 상단 헤더는 **현재 화면과 상관없이 전역적으로 영향을 미치는 동적 필터 및 유저 액션**을 배치합니다.

- **좌측:** 로고 (`Global CS Team` 전사 로고 혹은 G-Nexus 브랜드 로고)
- **중앙 (Global Filter & Search):**
  - 🗓️ **Date Range Picker:** 대시보드 및 데이터 보드 전체에 적용되는 [회계연도/분기/월간] 기간 선택기. (헤더에 둠으로써 매번 페이지마다 기간을 설정할 필요를 잃음)
  - 🔍 **Command Palette (검색):** `Ctrl+K` 로 호출가능한 통합 검색 바 (마크다운 분석 결과나 특정 문서의 지표 검색용).
- **우측 (User & Context):**
  - 🔔 **알림(Notifications):** Weak Signal (이상 징후 예측) 알림 센터.
  - 👤 **유저 프로필:** 내 정보, 권한(SSO Role), 다크모드/라이트모드 토글 스위치.

### 🟢 2-3. 메인 콘텐츠 영역 (Main Content)
핵심 컴포넌트들이 렌더링되는 영역입니다. 이 영역은 "페이지 타입"에 따라 두 가지 다른 뷰포트 전략을 취해야 합니다.

- **View Type 1. Dashboard 형태:** 
  - 화면 정가운데 위주로 배치되며, 모서리에 적절한 여백(Padding)이 있는 편안한 카드 레이아웃.
- **View Type 2. Data Grid (데이터 표) 형태:**
  - 엑셀과 같이 방대한 데이터를 봐야하므로, **여백을 0에 가깝게 최소화 (Full Width / Full Height)**하여 화면의 1픽셀까지 데이터 표시에 집중.

### 🟡 2-4. 하단 영역 (Footer) - [제외 권장]
엔터프라이즈 데이터 다루는 B2B 웹사이트에서는 저작권이나 회사 소개를 담는 일반적인 형태의 Footer는 **데이터 가독성을 해치기 때문에 존재하지 않는 것이 좋습니다.** 
필요한 경우 좌측 사이드바 최하단에 작게 버전 정보(e.g., `v1.0.0-poc`)만 노출하는 것을 권장합니다.

## 3. 요약 (User Review Required)

- **결론:** 불필요한 하단 Footer를 제거한 **[좌측 축소형 사이드바 + 상단 전역 필터 헤더 + 고밀도 메인 뷰]**의 3단 구조가 G-Nexus에 가장 이상적입니다.
- 이 구조(특히 사이드바 메뉴 명칭이나 상단 헤더의 전역 달력 필터)에 동의하신다면, 이 청사진을 바탕으로 `App.tsx` 및 `Layout.tsx`의 뼈대를 즉시 생성하겠습니다!
